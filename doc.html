Implementation

  • Technologies:

    • Front end - HTML, CSS, Bootstrap, Ajax

    • Backed end - Python (Django)

    • Database system - MySQL

  • File structure:

    • ch/ - contains views for processing the search field and a bucket for tickets.

    • conference_hub/ - main folder with setting.py, wsgi.py and asgi.py files.

    • templates/ - contains all html files.

    • static/ - contains css/js files.

    • media/ contains images.

    • conferences/

      • models/ - contains models for conferences and their events (lunch and lecture).
      • views/
        1. conferences.py - contains views for display, create, modify, edit and delete conferences.
        2. events.py - contains views for display, create, modify, edit and delete events.
      • forms/ - contains forms for create and edit conferences/lectures/lunches.

    • users/

      • models/ - contains models for conferenceuser (each registered user), organization, researcher, profile and organization-employee.
      • views/
        1. conferenceuser.py - contains views for sign up, log out and sign in users.
        2. organization.py - contains views for emplyees of organization.
        3. profile.py - contains views for update and delete account, change password.
        4. researcher.py - contains views for organization members'.
      • forms/ - contains forms for sign up users/organizations/researchers anf profile update.

  • Deploy:

    •  To deploy our web application we used pythonanywhere.com It is an online integrated development environment and web hosting service.
     Reasons why we choose it:
    • It provides WSGI-based web hosting for Django.
    • Easy set up a fully functional web server within a couple of clicks.
    • It provides 100 free CPU-seconds per day.

Installation

Before start you should install Python 3.10.6 and MySQL server 8.0.31.

mysql -u root -p - launch the mysql shell and enter it as the root user.

  • create database conference_hub; - create database.
  • exit - go back to the shell.

python3 -m venv <path_to_env> && source <path_to_env>/bin/activate - create new virtual environment and activate it.

pip3 install -r requirements.txt - install Python libraries using requirements.txt.

python3 manage.py makemigrations - create new migrations based on your models.

python3 manage.py migrate - apply your migrations from the previous step.

python3 manage.py collectstatic - collect all static files in one place.

python3 manage.py runserver - run server.
